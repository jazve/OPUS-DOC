## 案例一：数据科学专家

### 需求背景
构建一个专业的数据科学专家智能体，能够为企业提供数据分析、机器学习建模和商业洞察服务。

### 设计分析

#### 专业领域识别
- **核心领域**：数据科学、机器学习、统计分析
- **目标用户**：企业数据分析师、业务决策者、产品经理
- **应用场景**：数据分析项目、机器学习模型设计、业务数据解读

#### 能力需求分析
- **分析能力**：数据探索、模式识别、统计推断
- **建模能力**：机器学习算法选择、模型评估优化
- **解释能力**：将技术结果转化为业务洞察
- **建议能力**：提供可行的数据驱动决策建议

## 案例二：法律咨询助手

### 需求背景
为法律服务机构构建专业的法律咨询助手，能够提供法律信息查询、案例分析和初步法律建议。

### 设计分析

#### 专业特点
- **高度专业性**：法律条文理解、案例分析、程序规范
- **严格准确性**：法律信息不能有错误或误导
- **谨慎表达**：明确区分法律信息和法律建议
- **持续更新**：法律法规频繁变化，需要及时更新

### 完整实现

应用场景：法律信息查询和初步咨询 - 专为此场景优化设计
</identity>

<architecture>

#### 根据应用领域选择
- **专业咨询类**：需要深度专业知识和分析能力
- **创作生产类**：重视创造性和输出质量
- **教育培训类**：强调引导性和个性化适应

#### 根据复杂度选择
- **简单应用**：参考基础案例的核心结构
- **中等复杂度**：参考完整案例的设计方法
- **高复杂度**：结合多个案例的设计模式

### 案例定制化方法

#### 需求映射
```
用户需求 → 案例选择 → 架构调整 → 功能定制 → 测试验证
```

#### 架构继承
```
基础架构 → 领域知识替换 → 技能模块调整 → 格式模块定制 → 工作流优化
```

### 最佳实践提取

#### 共性模式
1. **专业身份定位**：所有案例都有明确的专业身份
2. **结构化知识体系**：知识组织都采用分层分类方式
3. **格式模块定制**：都根据专业特点定制输出格式
4. **工作流程化**：都采用结构化的处理流程

#### 差异化设计
1. **知识深度**：根据专业复杂度调整知识深度
2. **交互方式**：根据用户特点设计交互模式
3. **输出风格**：根据专业要求调整输出风格
4. **约束机制**：根据专业规范设定约束条件
